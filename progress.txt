Claude Opus / Sonnet / Haiku in 4.x typically do, but not 100%. CLASSIFY-001 / CLASSIFY-002 (next two features) will need a defensive JSON parser that strips ``` json ... ``` fences and any leading/trailing prose before calling `JSON.parse`. Decision: don't put that parser here in prompts.ts -- prompts is a pure-text module with no parsing concerns. The right home is a `parseLlmJson(response: string)` helper in classifier.ts (or a new src/lib/ai/parse.ts) co-located with the other classification logic. Noted for next session.
- Verbatim-quote constraint trade-off: the drift prompt explicitly tells the model "If you cannot find a verbatim supporting quote in either side, return classification \"ambiguous\" with low confidence." This is intentionally pre-aligned with CITE-001's downstream behavior (verifyCitations strips an alert if the quote isn't a substring of the source) so the model is incentivized to be honest about uncertainty rather than fabricate a near-quote. The alternative -- letting the model guess and relying on CITE-001 to suppress -- would inflate `itemsSuppressed` (PIPE-002) and waste an LLM call per suppressed alert. Pre-alignment trades a small recall hit (some real drift will be classified as ambiguous because the model can't find a clean verbatim quote) for a much lower hallucination rate. PIPE-001 / PIPE-002 will be where this trade-off is empirically validated.
- Not using Anthropic's structured-output / JSON-mode feature (which would force the response to a JSON Schema). Two reasons: (a) as of @anthropic-ai/sdk v0.93.0 the top-level `response_format` option is still in beta and rolled out unevenly across models, so depending on it now would couple LLM-002 to a specific SDK / model combination that may shift; and (b) the prompt-instruction approach is portable -- if a future session swaps to a different LLM provider, the same prompt strings work without rewriting the response-shape contract. The defensive JSON parser planned for CLASSIFY-001 makes prompt-instructed JSON nearly as reliable as enforced JSON mode in practice.
- Did not add explanation-generation prompt despite the file's prior comment mentioning it. The drift template already produces an `explanation` field (2-4 sentences) per app_spec §5 Step 3, so a separate explanation prompt would be redundant. The LLM-002 spec only requires `relevanceClassification` + `driftClassification`. Removed the misleading "Explanation generation" line from the file's header comment to match what's actually exported.
